================================================================================
TECHNICAL ARCHITECTURE
================================================================================

RAG (Retrieval-Augmented Generation)
- LocalVectorStore: In-memory with cosine similarity
- Production: Pinecone or Weaviate
- Embeddings: OpenAI text-embedding-3-small (1536 dimensions)
- Semantic search: Top 5 documents, 0.7 similarity threshold
- Response time: <5 seconds for retrieval

LLM Integration
- Provider options: Claude 3.5 Sonnet or GPT-4
- Streaming responses for real-time UI
- Role-aware system prompts (5 roles)
- Few-shot learning with examples
- Context injection (user history, integrations, errors)

Chat System
- Multi-session management per user
- Searchable history (full-text search)
- Message persistence with timestamps
- Rating system (1-5 stars with emoji)
- Suggested questions on empty state
- Typing indicator with animation

Escalation Workflow
- Automatic detection of escalation triggers
- Frustration detection (high/medium/low levels)
- WhatsApp Business API integration
- SupportTicket generation with context
- Auto-categorization (integration/access/technical/data/onboarding/general)
- Webhook for incoming messages

Analytics
- Chat volume tracking (daily aggregation)
- Escalation rate calculation
- Response time metrics (P50, P95)
- Satisfaction tracking (avg rating)
- Event-based categorization
- 30-day and custom date range support

KEY FEATURES
================================================================================

1. Context Awareness Per User
   - Integration status (QB, Xero, Plaid: connected/not_connected)
   - Onboarding progress tracking
   - Previous ticket history
   - Action history
   - Current errors and exceptions
   - Role-based permissions

2. Role-Specific Responses
   - Admin: Full feature access, system configuration focus
   - Accountant: Transaction detail, reporting, compliance
   - CFO: High-level analytics, forecasting, strategy
   - Finance Manager: Team management, workflow automation
   - Viewer: Read-only, dashboard insights

3. Error Mapping System
   - Regex patterns for error code extraction
   - Quick-path solution mapping
   - Error categorization (integration/access/technical/data/other)
   - Prevention tips included in responses

4. Frustration-Based Escalation
   - Keyword detection (frustrated, confused, angry, urgent)
   - Frustration levels: low/medium/high
   - Auto-escalation on high frustration
   - Empathetic response opening based on level

5. Intelligent Search
   - Semantic search via vector embeddings
   - Integration-aware filtering (show only relevant docs)
   - Role-aware filtering (admin sees system docs)
   - Batch document indexing
   - Threshold-based result filtering

6. WhatsApp Integration
- Escalation via WhatsApp Business API
- Full context serialization for human agent
- Incoming message webhook handling
- Bidirectional conversation support
- Message templating for consistency

7. Session Management
- Create new sessions on demand
- Retrieve full chat history
- Search across all user sessions
- Session status tracking (active/resolved/escalated)
- Auto-save on each message

8. Analytics Dashboard
- Real-time metrics at /api/help-centre/analytics
- Customizable date ranges
- Metric selection (volume, escalation_rate, etc.)
- Daily aggregation
- Trend analysis (up/down/stable)

================================================================================
DEPLOYMENT OPTIONS
================================================================================

1. Express.js (Node.js)
   - Use HelpCentreRoutes.registerRoutes(app)
   - Port 3001 default
   - Full control over middleware
   - Docker-ready

2. Next.js (React)
   - Use dynamic API routes
   - pages/api/help-centre/[...route].ts
   - Serverless deployment
   - Vercel native support

3. Vector Database Options
   - Local (development): In-memory with cosine similarity
   - Pinecone (production): Fully managed, auto-scaling
   - Weaviate (self-hosted): Open source, fine-grained control

4. LLM Providers
   - Claude 3.5 Sonnet (recommended): Strong context understanding
   - GPT-4 (alternative): Excellent instruction following
   - Both support streaming for real-time UI

================================================================================
QUICK START
================================================================================

1. Install Dependencies
   npm install

2. Build CSS
   npm run build:css

3. Set Environment Variables (.env.local)
   OPENAI_API_KEY=sk_test_...
   CLAUDE_API_KEY=sk-ant-...
   WHATSAPP_PHONE_ID=...
   WHATSAPP_API_TOKEN=...

4. Start Development Server
   npm run dev

5. Test Endpoint
   curl -X POST http://localhost:3001/api/help-centre/chat \
     -H "Content-Type: application/json" \
     -d '{"userId":"test","message":"How do I connect QB?","userContext":{...}}'

6. Integrate into Dashboard
   Import HelpCentreChat component
   Add floating button with modal

7. Deploy to Production
   Set production environment variables
   Switch to Pinecone vector store
   Enable rate limiting
   Configure WhatsApp webhook
